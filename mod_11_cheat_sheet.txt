=== TOPIC: Knowledge Check ===
Prompt: What web computing models support thin-client architecture? Select all that apply.
Options:
  a. Client-browser computing
  b. Pure HTML Web with CGI
  c. Server-side Computing
  d. Silverlight
Answer: Client-browser computing, Server-side Computing (a, c)
---
Prompt: AJAX control is implemented in the ASP .Net environment as:
Options:
  a. an HTML control
  b. a user control
  c. a DLL library function
  d. a Web control
Answer: a Web control (d)
---
Prompt: A server control in ASP .Net application (Select all that apply):
Options:
  a. is an object that forms a component in an ASPX page
  b. must have an extension name .aspx
  c. is executed at the client side
  d. can take input and emit an event upon an input
Answer: is an object that forms a component in an ASPX page, is executed at the client side (a, c)
---
Prompt: A user control:
Options:
  a. is faster in execution than a DLL function
  b. is re-usable in an ASPX page
  c. is a patch on an ASPX page
  d. keeps user-related settings together
Answer: is re-usable in an ASPX page (b)
---
Prompt: Which of the following computing models can be considered to be in thick client architecture? Select all that apply.
Options:
  a. Pure HTML
  b. HTML 5
  c. Adobe Flash
  d. Silverlight
Answer: HTML 5, Silverlight (b, d)
