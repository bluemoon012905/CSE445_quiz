=== TOPIC: Knowledge Check ===
Prompt: The <appSettings> element in Web.config file are typically used for storing:
Options:
  a. URLs of remote services
  b. security settings
  c. policies and application parameter values
  d. URLs to remote servers
Answer: policies and application parameter values (c)
---
Prompt: What are the challenges for defining a global variable in Global.asax file? Select all that apply:
Options:
  a. Create global files that can coordinate with each other
  b. Allow class instances to read the global file
  c. Address the problem of simultaneous write on the variable
  d. Address the performance problem if lock mechanism is used
Answer: Create global files that can coordinate with each other, Allow class instances to read the global file, Address the problem of simultaneous write on the variable, Address the performance problem if lock mechanism is used (a, b, c, d)
---
Prompt: When session states are used, a Web server identifies a revisiting browser through an ID:
Options:
  a. issued by the client machine
  b. associated with the client browser
  c. issued by the server
  d. associated with the client TCP session
Answer: issued by the server (c)
---
Prompt: Where do you store the secure machine key generated for your ASP .Net application?
Options:
  a. In the Web.config file, in the element AppSettings
  b. In ApplicationStart()
  c. In the Web.config file, in the element system.web
  d. In SessionStart()
Answer: In the Web.config file, in the element system.web (c)
---
Prompt: How can you prevent Web application deadlock in the case of copying data from one disk file to another disk file?
Options:
  a. Open the two files in two different threads
  b. Close one file before opening the other
  c. Open the two files in random order
  d. Open one for read and the other for write
Answer: Close one file before opening the other (b)

Prompt: Consider the following code:

void Session_Start(object sender, EventArgs e) {
    Application["counter"] = Convert.ToInt32(Application["counter"]) + 1;
}

Answer the following:

What type of file in an ASPX application does this piece of code most likely belong to?
Options:
  a. Global.asax (application-level file)
  b. Web.config configuration file
  c. Default.aspx.cs page code-behind
  d. Machine.config
Answer: Global.asax (application-level file) (a)
Explanation: Session_Start lives in Global.asax so it can track sessions for the whole app.
